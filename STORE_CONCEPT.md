## 1. Vision

The Store is a **shared infrastructure library** for all FreeSynergy projects.
It provides three core services across every project:

1. **Package Registry** — A structured directory of packages with a standardized
   `manifest.toml` format. A "package" may be a deployment module (FSN), a runtime
   plugin (Wiki.rs), a language pack, or any other distributable unit.

2. **i18n Distribution** — A centralized collection of locale files in TOML format,
   structured consistently across all namespaces and consumable via the `store-sdk`.

3. **Shared Crates** — Reusable Rust libraries (`store-sdk`, `config-serializer`)
   importable by any FreeSynergy project as Cargo dependencies.

**The Store is project-agnostic by design.** FreeSynergy.Node (FSN) and Wiki.rs are
the first consumers, but the Store can serve any future FreeSynergy application with
no structural changes to the Store itself — only a new namespace directory.

## 2. Core Standards

These rules apply **universally** to every namespace, every project, every package type.
No exceptions.

| Rule | Detail |
|---|---|
| **TOML is canonical** | All manifests, catalogs, and locale files are stored as `.toml`. The `config-serializer` crate handles conversion to other formats on demand. |
| **`manifest.toml` everywhere** | Every package directory contains exactly one `manifest.toml`. No `plugin.toml`, `module.toml`, or `lang.toml` — always `manifest.toml`. |
| **`catalog.toml` everywhere** | Every namespace has a `catalog.toml` (auto-generated by CI). No `index.toml`. |
| **`i18n/` everywhere** | Locale files live in `i18n/`. No `lang/`, no `locale/`. |
| **TOML table hierarchy in i18n** | Locale files use native TOML tables (`[section] key = "value"`), not flat quoted strings (`"section.key" = "value"`). |
| **Dot-notation for key lookup** | At runtime, table hierarchy is flattened to `"section.key"` by `store-sdk`. Consumer code never changes. |
| **`[package]` header in every manifest** | All manifests share a common `[package]` block. Project-specific fields go in additional sections. |
| **Dot-separated categories** | `deploy.proxy`, `plugin.auth`, `i18n.locale` — never flat strings. |
| **English in all code** | All struct fields, comments, and variable names in Rust/YAML/TOML keys are English. |
| **Semver** | Every package declares a `version` following semver. |
| **OSI-approved licenses** | All packages in the main Store use OSI-approved licenses. |

## 4. Universal Package Manifest

Every package — regardless of type or namespace — has a `manifest.toml` with a
standard `[package]` header. Project-specific data goes in additional sections
**after** the `[package]` block.

```toml
# manifest.toml — required in every package directory

[package]
id          = "proxy/zentinel"           # "{category-path}/{name}" — unique within namespace
name        = "Zentinel"
version     = "0.3.0"                    # semver
category    = "deploy.proxy"             # see §5 Category System
description = "Pingora-based reverse proxy with automatic TLS, DNS challenge and KDL config"
license     = "Apache-2.0"              # SPDX identifier
author      = "FreeSynergy"
tags        = ["proxy", "tls", "pingora", "acme"]

[package.source]
website    = "https://github.com/zentinelproxy/zentinel"
repository = "https://github.com/zentinelproxy/zentinel"

[package.compat]
min_app_version = "0.1.0"               # minimum version of the consuming application
projects        = ["Node"]              # which namespaces this package targets

# ── Project-specific sections follow ─────────────────────────────────────────

# FSN deployment module: [deploy] section
[deploy]
type        = "proxy"
constraints = { per_host = 1, per_ip = 1 }
port        = 443
```

### 4.1 i18n Package Manifest

Language packages use `category = "i18n.locale"` and add an `[i18n]` section:

```toml
[package]
id          = "i18n/de"
name        = "Deutsch"
version     = "1.0.0"
category    = "i18n.locale"
description = "German translation for FreeSynergy.Node"
license     = "Apache-2.0"
author      = "FreeSynergy"
tags        = ["i18n", "de", "german"]

[package.compat]
min_app_version = "0.1.0"
projects        = ["Node"]

[i18n]
locale_code  = "de"
native_name  = "Deutsch"
direction    = "ltr"                    # "ltr" or "rtl"
completeness = 100                      # 0–100, updated by CI
api_version  = 1                        # must match TRANSLATION_API_VERSION in app
```

It lists all packages **and** all locales. Never edit manually.

## 7. i18n Standard

### 7.1 Directory Structure

Every locale is a subdirectory, not a flat file. Minimum: `manifest.toml` + `ui.toml`.

```
{Namespace}/i18n/
├── en/
│   ├── manifest.toml      ← [package] + [i18n] (see §4.1)
│   └── ui.toml            ← Required: all UI strings
├── de/
│   ├── manifest.toml
│   └── ui.toml
└── ar/                    ← RTL example
    ├── manifest.toml      ← [i18n] direction = "rtl"
    └── ui.toml
```

Web apps (e.g. Wiki.rs) may add optional files:
```
Wiki.rs/i18n/de/
├── manifest.toml
├── ui.toml
├── emails.toml            ← Email templates (Jinja2 interpolation supported)
└── errors.toml            ← Backend error messages
```

### 7.2 `ui.toml` Format

Locale files use **native TOML table hierarchy** — never flat quoted string keys.
The `[meta]` section at the top makes the file self-contained for offline use.

```toml
# ui.toml — TOML table hierarchy, NOT quoted flat strings

[meta]
code         = "de"
name         = "Deutsch"
api_version  = 1
completeness = 100

[welcome]
title    = "Willkommen bei FreeSynergy.Node"
subtitle = "Dezentrale Infrastruktur — frei und selbst betrieben"

[common]
save   = "Speichern"
cancel = "Abbrechen"

[sys]
host    = "Host"
user    = "Benutzer"
ip      = "IP-Adresse"
ram     = "Arbeitsspeicher"
cpu     = "CPU-Kerne"
uptime  = "Laufzeit"
podman  = "Podman"
arch    = "Architektur"

[field_help]
name   = "Eindeutiger Kurzname. Nur Kleinbuchstaben, Ziffern und Bindestriche."
domain = "Primäre Domain ohne http://. Wildcards erlaubt: *.example.com"
```

**Key rules:**
- Top-level sections match UI component hierarchy
- The section name becomes the prefix when flattened to dot-notation: `[welcome] title` → `"welcome.title"`
- `[meta]` and `[field_help]` are reserved section names
- `[field_help]` keys are single-level (no nesting) — used for contextual help text
- Required top-level sections (must be consistent across all locales):
  `[common]`, `[welcome]`, `[sys]`, `[nav]`, `[form.*]`, `[status]`, `[settings.*]`

### 7.3 Email Templates (Jinja2)

```toml
# emails.toml — Wiki.rs only

[welcome]
subject = "Willkommen bei {{ site_name }}, {{ user_name }}!"
body = """
Hallo {{ user_name }},

dein Konto auf {{ site_name }} wurde erstellt.
Anmelden unter: {{ login_url }}

Viele Grüße,
Das {{ site_name }}-Team
"""
```

### 7.4 Completeness Tracking (CI)

The `validate-i18n.yml` workflow:
1. Loads `en/ui.toml` as the reference key set.
2. For every other locale, counts matching keys.
3. Updates `completeness` in `manifest.toml` and `catalog.toml` automatically.
4. Warns (but does not fail) if completeness < 70%.
5. Fails if `direction = "rtl"` is missing for known RTL locales: `ar`, `fa`, `he`, `ur`, `ug`, `ps`.

## 8. `store-sdk` Crate (Trait-based)

Published as `freesy-store-sdk`. The SDK is **fully generic** — projects supply
their own manifest types; the SDK provides catalog infrastructure and i18n loading.

### 8.1 `Manifest` Trait

```rust
/// Every package type must implement this trait.
pub trait Manifest: for<'de> serde::Deserialize<'de> + Send + Sync {
    fn id(&self) -> &str;
    fn version(&self) -> &str;
    fn category(&self) -> &str;
    fn package_meta(&self) -> &PackageMeta;
}

/// The [package] section present in every manifest.toml.
#[derive(Debug, Clone, serde::Deserialize)]
pub struct PackageMeta {
    pub id:          String,
    pub name:        String,
    pub version:     String,
    pub category:    String,
    pub description: String,
    pub license:     String,
    pub author:      String,
    #[serde(default)]
    pub tags:        Vec<String>,
    pub source:      Option<PackageSource>,
    pub compat:      Option<PackageCompat>,
}

#[derive(Debug, Clone, serde::Deserialize)]
pub struct PackageSource {
    pub website:    Option<String>,
    pub repository: Option<String>,
}

#[derive(Debug, Clone, serde::Deserialize)]
pub struct PackageCompat {
    pub min_app_version: Option<String>,
    #[serde(default)]
    pub projects: Vec<String>,
}
```

### 8.2 `Catalog<M>`

```rust
/// Generic catalog — works for any manifest type.
pub struct Catalog<M: Manifest> {
    entries: Vec<M>,
}

impl<M: Manifest> Catalog<M> {
    /// List all packages.
    pub fn list(&self) -> &[M];

    /// Find by ID.
    pub fn find(&self, id: &str) -> Option<&M>;

    /// Filter by category prefix (e.g. "deploy." matches all deploy.* categories).
    pub fn by_category(&self, prefix: &str) -> Vec<&M>;

    /// Load from a catalog.toml via a StoreClient.
    pub fn load(client: &StoreClient, toml_key: &str, path: &Path) -> Result<Self>;
}
```

### 8.3 `StoreClient`

```rust
pub struct StoreClient {
    source: StoreSource,
}

pub enum StoreSource {
    /// Local filesystem (git clone, submodule, or dev path)
    LocalPath(PathBuf),
    /// Remote git repository — cloned on first use, cached locally
    RemoteGit { url: String, branch: String, cache_dir: PathBuf },
}

impl StoreClient {
    pub fn new(source: StoreSource) -> Self;

    /// Load and deserialize a Catalog from a TOML file.
    /// `toml_key` is the TOML array key, e.g. "packages" or "modules".
    pub fn load_catalog<M: Manifest>(&self, toml_key: &str, path: &Path) -> Result<Catalog<M>>;

    /// Load the full i18n bundle for a locale.
    pub fn load_i18n(&self, namespace: &str, locale_code: &str) -> Result<I18nBundle>;

    /// Low-level: load any file and return as TomlValue.
    pub fn load_toml(&self, relative_path: &Path) -> Result<toml::Value>;
}
```

### 8.4 `I18nBundle`

```rust
pub struct I18nBundle {
    pub meta:   I18nMeta,
    pub ui:     toml::Value,
    pub emails: Option<toml::Value>,
    pub errors: Option<toml::Value>,
}

#[derive(Debug, Clone, serde::Deserialize)]
pub struct I18nMeta {
    pub code:         String,
    pub name:         String,
    pub direction:    TextDirection,
    pub completeness: u8,
    pub api_version:  u32,
}

#[derive(Debug, Clone, serde::Deserialize, PartialEq)]
#[serde(rename_all = "lowercase")]
pub enum TextDirection { Ltr, Rtl }

impl I18nBundle {
    /// Returns the ui TomlValue as-is (zero-copy).
    pub fn as_toml(&self) -> &toml::Value;

    /// Converts ui to JSON string (for Vue i18n, REST API, etc.).
    pub fn to_json(&self) -> Result<String>;

    /// Converts ui to YAML string (for config export, etc.).
    pub fn to_yaml(&self) -> Result<String>;

    /// Flattens the TOML table hierarchy to dot-notation HashMap.
    /// `[welcome] title = "..."` → `"welcome.title"`.
    /// Used by terminal UIs, static lookups, and anywhere needing t("key").
    pub fn to_hashmap(&self) -> HashMap<String, String>;
}
```

### 8.5 Project-Specific Manifest Types

Projects define their own manifest types in their own crates, not in `store-sdk`.

**FSN** (`fsn-core`):
```rust
// Replaces StoreEntry — implements Manifest trait
#[derive(Debug, Clone, serde::Deserialize)]
pub struct ModuleManifest {
    pub package: PackageMeta,
    pub deploy:  Option<DeployMeta>,
}

impl Manifest for ModuleManifest {
    fn id(&self)           -> &str { &self.package.id }
    fn version(&self)      -> &str { &self.package.version }
    fn category(&self)     -> &str { &self.package.category }
    fn package_meta(&self) -> &PackageMeta { &self.package }
}

pub type ModuleCatalog = Catalog<ModuleManifest>;
```

**Wiki.rs** (`wiki-core`):
```rust
#[derive(Debug, Clone, serde::Deserialize)]
pub struct PluginManifest {
    pub package: PackageMeta,
    pub plugin:  Option<PluginMeta>,
}
impl Manifest for PluginManifest { ... }

pub type PluginCatalog = Catalog<PluginManifest>;
```

## 9. `config-serializer` Crate

Published as `freesy-config-serializer`. Fully independent — no FSN or Wiki.rs
specific code. Any FreeSynergy project imports it directly.

```toml
[dependencies]
freesy-config-serializer = "1.0"
```

### 9.1 Format Support Matrix

| Format | Read | Write | Crate |
|---|---|---|---|
| TOML | ✅ | ✅ | `toml` |
| YAML | ✅ | ✅ | `serde_yaml` |
| JSON | ✅ | ✅ | `serde_json` |
| INI | ✅ | ✅ | `ini` |
| Jinja2 | ✅ (render) | ✅ (generate) | `minijinja` |
| CSV | ✅ (key-value) | ✅ | `csv` |

### 9.2 API

```rust
pub struct ConfigSerializer;

impl ConfigSerializer {
    /// Load any supported format; auto-detect from file extension.
    pub fn load(path: &Path) -> Result<toml::Value, ConfigError>;

    /// Save as any supported format; auto-detect from file extension.
    pub fn save(value: &toml::Value, path: &Path) -> Result<(), ConfigError>;

    /// Parse a string in a known format.
    pub fn from_str(input: &str, format: ConfigFormat) -> Result<toml::Value, ConfigError>;

    /// Serialize to a string in a known format.
    pub fn to_string(value: &toml::Value, format: ConfigFormat) -> Result<String, ConfigError>;

    /// Convert directly between two formats.
    pub fn convert(input: &str, from: ConfigFormat, to: ConfigFormat) -> Result<String, ConfigError>;
}

#[derive(Debug, Clone, Copy, PartialEq)]
pub enum ConfigFormat { Toml, Yaml, Json, Ini, Jinja2, Csv }
```

### 9.3 Auto-detection by Extension

| Extension | Format |
|---|---|
| `.toml` | TOML |
| `.yaml`, `.yml` | YAML |
| `.json` | JSON |
| `.ini`, `.cfg`, `.conf` | INI |
| `.jinja2`, `.j2`, `.jinja` | Jinja2 |
| `.csv` | CSV |

### 9.4 Internal Representation

All formats are converted to/from `toml::Value` as the canonical internal type.
TOML features (tables, arrays, datetimes) are preserved without loss.

## 10. Project Namespaces

The Store supports multiple FreeSynergy projects in the same repository via
top-level namespace directories:

```
Store/
├── Node/         ← FreeSynergy.Node — deployment modules + TUI i18n
├── Wiki.rs/      ← Wiki.rs — runtime plugins + full i18n
├── Forum.rs/     ← (future) Forum project
└── shared/       ← Packages usable by multiple projects
```

A package in `shared/` is referenced from any project's `catalog.toml`:

```toml
[[packages]]
id       = "auth/kanidm"
category = "shared.auth"
path     = "shared/plugins/auth-kanidm"
```

**Override mechanism for i18n:** If `Wiki.rs/i18n/de/ui.toml` exists, it takes
precedence over `shared/i18n/de/ui.toml` for Wiki.rs-specific keys.

## 11. Store API (optional HTTP layer)

For deployments that want to serve packages over HTTP, an optional `store-server`
binary is included. It is a lightweight `axum` service that serves the Store
directory.

```
GET /api/v1/{namespace}/catalog                    → catalog.toml (JSON)
GET /api/v1/{namespace}/packages                   → [[packages]] as JSON
GET /api/v1/{namespace}/packages/{id}              → manifest.toml (JSON)
GET /api/v1/{namespace}/packages/{id}/download     → package bundle
GET /api/v1/{namespace}/i18n                       → [[locales]] from catalog (JSON)
GET /api/v1/{namespace}/i18n/{code}                → ui.toml as JSON
GET /api/v1/{namespace}/i18n/{code}/emails         → emails.toml as JSON
GET /api/v1/{namespace}/i18n/{code}/errors         → errors.toml as JSON
```

All endpoints return TOML converted to JSON via `ConfigSerializer`.

## 12. Consuming the Store

### 12.1 FreeSynergy.Node (FSN) — as git submodule or HTTP

```toml
# ~/.config/fsn/settings.toml
[[stores]]
name       = "FSN Official"
url        = "https://raw.githubusercontent.com/FreeSynergy/Store/main"
git_url    = "https://github.com/FreeSynergy/Store.git"
enabled    = true
```

FSN fetches `{url}/Node/catalog.toml` for the module and locale index.

```rust
// fsn-engine usage
let catalog = store.load_catalog::<ModuleManifest>("packages", Path::new("Node/catalog.toml"))?;
let proxy_modules = catalog.by_category("deploy.proxy");

// i18n loading
let bundle = store.load_i18n("Node", "de")?;
let map = bundle.to_hashmap();   // → HashMap<String, String> for t("welcome.title")
```

### 12.2 Wiki.rs — as Cargo dependency + git submodule

```toml
# Cargo.toml
[dependencies]
freesy-store-sdk         = { git = "https://github.com/FreeSynergy/Store" }
freesy-config-serializer = { git = "https://github.com/FreeSynergy/Store" }
```

```rust
// wiki-core usage
let store = StoreClient::new(StoreSource::LocalPath(PathBuf::from("./store")));

let plugins = store.load_catalog::<PluginManifest>("packages", Path::new("Wiki.rs/catalog.toml"))?;
let auth_plugins = plugins.by_category("plugin.auth");

// i18n for Vue i18n frontend
let bundle = store.load_i18n("Wiki.rs", "de")?;
let json = bundle.to_json()?;    // → JSON string for vue-i18n
```

### 12.3 Future Projects

## 13. Governance & Contribution

### 13.1 Adding a New Package

1. Create `{Namespace}/{category-path}/{name}/manifest.toml` following §4.
2. Implement project-specific sections (`[deploy]`, `[plugin]`, etc.).
3. Open a PR — CI validates the manifest schema and license.
4. On merge, `publish-catalog.yml` regenerates `catalog.toml` automatically.

### 13.2 Adding a New Locale

1. Create `{Namespace}/i18n/{code}/manifest.toml` with `[package]` + `[i18n]`.
2. Create `ui.toml` using TOML table hierarchy. Use `en/ui.toml` as template.
3. Open a PR — CI checks completeness, RTL flag, and api_version.
4. On merge, `validate-i18n.yml` updates `completeness` in `manifest.toml` and `catalog.toml`.

### 13.3 Adding a New Project Namespace

1. Create `{ProjectName}/` directory with `catalog.toml` (initially empty).
2. Add `modules/` or `plugins/` and `i18n/` subdirectories.
3. Follow all §2 Core Standards.
4. CI pipelines apply automatically via glob patterns.

### 13.4 Versioning Policy

- Package versions follow semver. Breaking API changes require a major version bump.
- i18n package versions follow `{major}.{minor}.{patch}` — patch increments on any key addition.
- `catalog.toml` is auto-generated — never edit manually.

### 13.5 License Policy

All contributions to the main Store must be under an OSI-approved open-source
license. Organizations may fork the Store for private packages.

### 13.6 CI Workflows

| Workflow | Trigger | Action |
|---|---|---|
| `validate-packages.yml` | PR | Validate all `manifest.toml` files |
| `validate-i18n.yml` | PR | Check completeness, RTL flags, api_version |
| `publish-catalog.yml` | push to main | Regenerate all `catalog.toml` files |
